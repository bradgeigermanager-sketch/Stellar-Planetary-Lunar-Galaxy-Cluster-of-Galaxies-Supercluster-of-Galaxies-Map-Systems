## Documentation Style

Each schema page should use the same template:

```

Hierarchy Position

Description

Properties

Validation

Example

Related Schemas

Revision History
```

This way every page has a familiar structure.

## Mermaid Diagrams

GitHub Pages supports Mermaid, which makes the documentation much easier to follow.

Example:

````markdown
```mermaid
graph TD
    SC[Supercluster]
    GC[Galaxy Cluster]
    G[Galaxy]
    SS[Star System]
    S[Star]
    P[Planet]
    M[Moon]

    SC --> GC
    GC --> G
    G --> SS
    SS --> S
    SS --> P
    P --> M
```
````

No images need to be maintained—the diagram is generated from the text.

Future Growth Directions

This is a standards project with documents beyond the schema references:

```
docs/

specification.md
architecture.md
roadmap.md
contributing.md
governance.md
license.md
```

These documents help contributors understand not just what the schemas define, but why they are designed that way and how changes should be proposed.

This structure scales well from a handful of schemas to a much larger specification without requiring major reorganization later.
